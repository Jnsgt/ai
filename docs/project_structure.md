# Project Structure

This repository separates reusable package code, example assets, scripts, and tests.

- `cldltest/`: core package, CLI, runners, metrics, and comparison logic
- `docs/`: project-level documentation
- `examples/`: small demo-ready inputs, models, and sample outputs
- `models/`: model definitions and ONNX export helpers
- `scripts/`: GUI or local helper scripts
- `tests/`: standalone testing utilities

This layout keeps the repository easier to read and more suitable for open-source collaboration.
